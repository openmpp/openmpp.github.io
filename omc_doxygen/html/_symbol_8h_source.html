114 // TODO: Explain better the use of double indirection to persist of Symbols and allow morphing within the SYmbol class hierarchy.
115 // When they need to be used, Symbol*& is used to get the underlying object.
116 // That lets us morph an object to another type during / parsing, while maintaining
117 // the persistence of symbol uses in other objects.
118 // A rule must be followed, which is to never store a pointer to a Symbol, either in a Symbol or another object.
119 // Instead, store a reference to the pointer of, which can always be obtained using the member function stable_rp().
120 // Actually, it's a reference to the second pair of the element for the symbol in the symbol table (the first element of the pair is the name)
121 // The underlying collection std::unordered_map guarantees that references and pointers remain valid even if rehashing occurs.
