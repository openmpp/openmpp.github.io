112 // TODO: Explain better the use of double indirection to persist of Symbols and allow morphing within the SYmbol class hierarchy.
113 // When they need to be used, Symbol*& is used to get the underlying object.
114 // That lets us morph an object to another type during / parsing, while maintaining
115 // the persistence of symbol uses in other objects.
116 // A rule must be followed, which is to never store a pointer to a Symbol, either in a Symbol or another object.
117 // Instead, store a reference to the pointer of, which can always be obtained using the member function stable_rp().
118 // Actually, it's a reference to the second pair of the element for the symbol in the symbol table (the first element of the pair is the name)
119 // The underlying collection std::unordered_map guarantees that references and pointers remain valid even if rehashing occurs.
